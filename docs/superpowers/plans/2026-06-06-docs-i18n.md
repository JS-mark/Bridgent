# apps/docs 中英双语化 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 `apps/docs`(VitePress)从单语英文站点扩展为「英文(默认)+ 简体中文(`/zh/` 前缀)」的双语站点,内容全量翻译。

**Architecture:** 采用 VitePress 原生 `locales` 配置,不引入任何第三方 i18n 插件。把 `themeConfig` 按语言拆分到 `.vitepress/locales/{en,zh,shared}.ts` 三个文件,主 `config.ts` 只做装配。中文内容镜像放在 `apps/docs/zh/`,文件名与英文版一一对应。

**Tech Stack:** VitePress 1.x、TypeScript、pnpm workspace、Turborepo。

**Source spec:** `docs/superpowers/specs/2026-06-06-docs-i18n-design.md`(执行前先通读,术语表与翻译规范在第 5 节)。

**Pre-flight notes:**
- 仓库当前可能尚无任何 commit。所有 `git add` 命令都使用**显式路径**,不要使用 `-A` 或 `.`,避免误把本任务无关的 untracked 文件拉进来。
- 验证命令优先用 `pnpm --filter @bridgent/docs <script>` 直接跑(快),Turbo 全套留到最后一步。
- 不要修改任何英文 markdown 内容(仅可能因 `link` 路径已经是裸路径而无需调整)。

## Task 4: 改写 `.vitepress/config.ts`,装配 locales

- [ ] **Step 1: 重写 config.ts**

```ts
// apps/docs/.vitepress/config.ts
import { defineConfig } from 'vitepress'
import { en } from './locales/en'
import { shared } from './locales/shared'
import { zh } from './locales/zh'

export default defineConfig({
  ...shared,
  themeConfig: {
    ...shared.themeConfig,
  },
  locales: {
    root: { label: 'English', lang: 'en-US', ...en },
    zh: { label: '简体中文', lang: 'zh-CN', ...zh },
  },
})
```

注意:`themeConfig` 显式合并一次是因为 `shared` 中含 `themeConfig.socialLinks`,直接 spread 覆盖会丢掉。VitePress 会进一步把 root locale 的 `themeConfig`(来自 `en.themeConfig`)与顶层 `themeConfig` 合并。
