## Goals

## Non-goals

- No `bridgent expose`.
- No OAuth2 / PKCE.
- No Prisma write tools in runtime code.
- No raw SQL exposure.
- No custom Inspector web UI.
- No hidden config file or project model.
- No tRPC or GraphQL adapter in this iteration; Drizzle is the chosen v0.2 source expansion.

## OpenAPI API-key auth

Support:

```ts
auth: { type: 'apiKey', in: 'header' | 'query' | 'cookie', name: string, value: string | (() => string | Promise<string>) }
```

Behavior:

- `header`: set the configured header.
- `query`: append/set the configured query parameter.
- `cookie`: append to the `Cookie` header.
- Bearer auth remains unchanged.

## Drizzle source adapter

Package: `@bridgent/source-drizzle`.

Initial scope:

- Read-only `findMany` tool per table.
- User passes an instantiated Drizzle db and a table map.
- No raw SQL, no writes, no generated config.
- Limit is clamped by `maxLimit`; default limit is used when omitted.
- Optional offset.
- Tool result envelope mirrors existing source packages: `{ ok, result, meta, error }`.

The adapter is deliberately structural and light on Drizzle internals. It calls the common Drizzle read builder shape:

```ts
db.select().from(table).limit(n).offset(n)
```

- destructive operation guardrails

Runtime write tools remain non-goal until the design is reviewed.
